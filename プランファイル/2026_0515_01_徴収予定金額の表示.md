# 2026_0515_01 徴収予定合計金額の表示

## Context（背景・目的）

徴収チェック画面（`/Collections/Details/{id}`）では現在、画面上部に「徴収済合計／徴収済／未徴収／除外」の4つのサマリーカードが表示されている。
一方で、**「除外（不参加）」を除いた全員分の徴収予定合計金額（=本来集めるべき金額）** が一目で分からない。

本対応では以下を実現する：

- 徴収チェック画面に「徴収予定金額」の合計を表示
- 「除外（不参加）」チェックが切り替わった際に、画面リロードなしで徴収予定合計金額のみを自動再計算
- 既存サマリーカードは右寄せに整列し、左側に「徴収予定金額」カードを新設

## 実装内容

### 1. ViewModel への追加

`LPG_Tsumitate_Kanri2/Models/ViewModels/CollectionCheckViewModel.cs`

```csharp
public class CollectionCheckViewModel
{
    public CollectionSession Session { get; set; } = null!;
    public List<CollectionRecordRow> Records { get; set; } = new();
    public int TotalExpected { get; set; }     // 追加: 徴収予定金額（除外を除いた全員の合計）
    public int TotalCollected { get; set; }
    public int CollectedCount { get; set; }
    public int PendingCount { get; set; }
    public int ExcludedCount { get; set; }
    public bool IsNormalSavings => Session.SavingsTypeId == 1;
}
```

### 2. Controller への集計ロジック追加

`LPG_Tsumitate_Kanri2/Controllers/CollectionsController.cs` の `Details` アクション（93-131行目）の ViewModel 構築箇所に1行追加：

```csharp
var vm = new CollectionCheckViewModel
{
    Session = session,
    Records = session.CollectionRecords
        .OrderBy(r => r.Employee.EmployeeNo)
        .Select(r => new CollectionRecordRow { /* 既存のまま */ }).ToList(),
    TotalExpected = session.CollectionRecords
        .Where(r => !r.IsExcluded)
        .Sum(r => r.ExpectedAmount),                                                     // 追加
    TotalCollected = session.CollectionRecords.Where(r => r.IsCollected && !r.IsExcluded).Sum(r => r.ExpectedAmount),
    CollectedCount = session.CollectionRecords.Count(r => r.IsCollected && !r.IsExcluded),
    PendingCount = session.CollectionRecords.Count(r => !r.IsCollected && !r.IsExcluded),
    ExcludedCount = session.CollectionRecords.Count(r => r.IsExcluded)
};
```

「徴収予定金額」の定義 = **除外されていない全員分の `ExpectedAmount` の合計**。
（除外＝不参加なので集めない、徴収済／未徴収はどちらも「予定」に含まれる。）

**After（変更後）**: 左に「徴収予定金額」カード、右側に既存4枚を寄せる。

### 4. updateSummary() の拡張

`LPG_Tsumitate_Kanri2/Views/Collections/Details.cshtml` の207-219行目の `updateSummary()` 関数に「徴収予定金額」の再計算ロジックを追加：

```javascript
function updateSummary() {
    let totalExpected = 0;                   // 追加
    let total = 0, collected = 0, pending = 0, excluded = 0;
    document.querySelectorAll('.check-toggle').forEach(cb => {
        if (cb.disabled) { excluded++; return; }       // disabled = 除外済み
        totalExpected += parseInt(cb.dataset.amount);  // 追加: 除外以外を合計
        if (cb.checked) { total += parseInt(cb.dataset.amount); collected++; }
        else pending++;
    });
    const expEl = document.getElementById('totalExpected');
    if (expEl) expEl.textContent = totalExpected.toLocaleString() + ' 円';   // 追加
    document.getElementById('totalAmount').textContent = total.toLocaleString() + ' 円';
    document.getElementById('collectedCount').textContent = collected + ' 人';
    document.getElementById('pendingCount').textContent = pending + ' 人';
    const excEl = document.getElementById('excludedCount');
    if (excEl) excEl.textContent = excluded + ' 人';
}
```

判定キー：`cb.disabled` が true = 除外済み（既存実装に従う）。除外されていないチェックボックスの `data-amount` を合計する。

### 5. 既存 Ajax フローとの連動（追加実装不要）

除外チェック切替時のフロー（`Details.cshtml` 165-191行目）はすでに以下を実施しており、徴収予定合計金額の自動更新は `updateSummary()` の拡張だけで達成される：

1. `.exclude-toggle` の change イベントで `POST /Collections/Exclude` を Ajax 送信
2. レスポンス成功時に該当行のチェックボックスを `disabled` ←→ `enabled` トグル
3. 最後に `updateSummary()` を呼び出して全サマリーを再集計

→ 拡張した `updateSummary()` が自動で `totalExpected` も更新するため、追加実装は不要。

### 6. 締切済みセッションの扱い

既存サマリー（徴収済合計など）は `Session.IsCompleted` の状態に関わらず常に表示されている。「徴収予定金額」も同じく常時表示で問題なし。締切済みの場合は除外チェックボックス自体が表示されないため、再計算イベントも発生しない（=初期表示の値が確定値となる）。

## 検証手順

ローカルビルド・実行は作業者（ユーザー）が行う。Claude 側ではビルド・起動は行わない。

1. `dotnet build` でビルドエラーなしを確認
2. `dotnet run --project LPG_Tsumitate_Kanri2/LPG_Tsumitate_Kanri2.csproj` で起動
3. ブラウザで `https://localhost:7084/Collections/Details/{任意のセッションID}` を開く
4. 画面上部に「徴収予定金額 ◯◯◯ 円」が左側に表示され、右側に既存4枚のサマリーカードが並ぶこと
5. 初期表示値が「除外されていない社員の ExpectedAmount の合計」と一致すること（手計算で検証）
6. 任意の社員の「除外（不参加）」チェックを ON にすると：
   - 画面リロードが発生しないこと
   - 「徴収予定金額」がその社員の予定金額だけ減算されること
   - 「除外」人数が +1 されること
7. 同じチェックを OFF に戻すと、徴収予定金額が元の値に戻ること
8. スマホ表示（DevTools のレスポンシブモード / iPhone 等）でレイアウトが縦並びに折り返すこと
9. 締切済みセッションでも「徴収予定金額」が正しく表示されること

## 補足

- 本対応はバックエンドのデータモデル変更を伴わない（マイグレーション不要）。
- ContributionCalculator や Ledger 系のロジックは触らない。
- 「徴収予定金額」の集計は Controller 側（サーバ初期表示）と View 側（クライアント動的更新）で二重に計算するが、これは既存の `TotalCollected` / `CollectedCount` 等と同じパターンに揃えたもの。
